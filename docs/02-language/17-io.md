# IO 与文件基础

够用即可。现代 Java 文件操作用 `java.nio.file` 的 `Files` / `Path`，**别去啃老式的 `InputStream`/`FileReader`**（那是历史包袱，Spring 里几乎用不到）。

## 核心 API

```java
import java.nio.file.*;

Path file = Paths.get("target", "demo.txt");

// 写文件
Files.write(file, List.of("第一行", "第二行"), StandardCharsets.UTF_8);

// 读文件（一次全部行）
List<String> lines = Files.readAllLines(file, StandardCharsets.UTF_8);

// 追加
Files.write(file, List.of("第三行"), StandardCharsets.UTF_8, StandardOpenOption.APPEND);
```

## ★ 受检异常

```java
public static void main(String[] args) throws IOException { ... }
```

文件操作抛 `IOException`（受检异常，第 11 章讲过）。要么 `try/catch`，要么 `throws` 声明。

## 前端人对照

| 场景 | 前端 | Java |
|---|---|---|
| 读文本 | `fs.readFile`（Node） | `Files.readAllLines` |
| 写文本 | `fs.writeFile` | `Files.write` |
| 路径 | `'a/b.txt'` 字符串 | `Path` 对象 |

## 实战提示

- 后端业务里，文件上传/导出会用到 IO。Spring 提供了 `MultipartFile` 封装，底层还是这些。
- 大文件别用 `readAllLines`（一次性读进内存），用 `Files.lines()` 返回 Stream 流式处理。
- 编码**永远显式指定 UTF-8**，避免中文乱码。
